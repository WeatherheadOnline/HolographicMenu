Mobile-friendly "input menu"

Improve your users' mobile experience for form inputs

Form input fields can be difficult to reach when you're using your mobile device one-handed. If you could just.. reach... a bit higher... with your thumb...

This handy input-menu tool brings each of your page's form inputs within easy reach of your users' thumb.

Just add the included class and property to each of your form inputs (see JS tab for instructions). The input menu automatically populates with a button for each form input. When the button is tapped, focus is placed on the matching form input.

Buttons are color-coded to direct the user to checkboxes, radio buttons, or text fields. Buttons' borders are color-coded to show which form they belong to (for forms with a unique name attribute).

The one downside of this tool is that the menu becomes hidden by the on-screen keyboard. In the future, there will be a way to fix this issue, after the VirtualKeyboard API gains wider browser support. (As a user, I still find it easier to use the input menu and then minimize the keyboard after each input, as both are within easy reach using my thumb.)
